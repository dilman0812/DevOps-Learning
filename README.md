# DevOps Learning Journey

This repository documents my structured, hands-on DevOps learning journey based on a phased roadmap, progressing from core fundamentals to production-grade DevOps practices.

---

## Learning Roadmap

### PHASE 1: FOUNDATION
**Goal:** Build strong fundamentals required for all DevOps tools

- **Linux Basics**
  - Linux architecture & file system
  - Essential commands
  - Permissions, users, processes
  - Shell basics

- **Networking Basics**
  - TCP/IP fundamentals
  - DNS, HTTP/HTTPS
  - Ports, firewalls, troubleshooting

- **Git & GitHub **
  - Git basics (clone, add, commit, push)
  - Branching & merging
  - GitHub workflows and best practices

---

### PHASE 2: CLOUD – AWS
**Goal:** Understand core cloud infrastructure used in DevOps

- EC2 (compute, security groups, key pairs)
- S3 (storage, lifecycle, permissions)
- IAM (users, roles, policies)
- Load Balancers (ALB concepts)

---

### PHASE 3: CI / CD PIPELINES
**Goal:** Automate build, test, and deployment workflows

- GitHub Actions
- Jenkins fundamentals
- CI/CD pipeline stages:
  - Build → Test → Deploy

---

### PHASE 4: CONTAINERS
**Goal:** Containerize applications and manage dependencies

- Docker fundamentals
- Dockerfile creation
- Docker Compose
- Image building & container lifecycle

---

### PHASE 5: KUBERNETES
**Goal:** Orchestrate containerized applications at scale

- Pods
- Deployments
- Services
- ConfigMaps & Secrets
- Basic troubleshooting

---

### PHASE 6: INFRASTRUCTURE AS CODE
**Goal:** Provision infrastructure using code

- Terraform basics
- Resource creation
- State management

---

### PHASE 7: MONITORING & SECURITY
**Goal:** Observe and secure production systems

- Prometheus & Grafana (metrics & dashboards)
- ELK Stack (logs & analysis)

---

## Learning Methodology

- Daily notes (Markdown)
- Hands-on commands and labs
- Shell scripts and automation
- Mini projects for each phase
- Daily Git commits for consistency

---

## Repository Structure

Each phase contains:
- Notes
- Commands practiced
- Scripts (where applicable)
- Interview questions & answers

---

## Goal

To become a ** DevOps Engineer** by gaining strong fundamentals, hands-on experience, and real-world DevOps exposure.
